# Case Study: Eliminated Critical Bugs in UK AI Meeting SaaS Before Release (NDA)

## Overview
- **Client type:** SaaS startup
- **Industry:** AI / Productivity (Online Meeting Platform, UK)
- **Service provided:** Manual QA Engineer — full-cycle QA setup, exploratory testing, cross-platform coverage
- **Tags:** Manual Testing, Exploratory Testing, Functional Testing, Regression Testing, Compatibility Testing, BrowserStack, Bug Reporting, QA Process Setup, Web Testing, UI/UX Testing

---

## The Challenge
A UK-based AI-powered meeting SaaS startup had no dedicated QA engineer and no formal testing process. Testing was handled ad hoc by the project manager, with no regression control or documentation in place.

Key pain points:
- No QA structure, test documentation, or checklists
- No regression control between sprints
- Multiple user roles and real-time collaboration features were untested
- Cross-platform and cross-browser coverage was non-existent
- The product was approaching release with unknown stability

---

## What I Did
- Conducted exploratory testing from scratch, uncovering **60+ previously undetected issues**, including **15 high-severity bugs** affecting user roles and real-time collaboration
- Built a **living test checklist** covering functional, regression, UI/UX, compatibility, and exploratory testing — maintained and updated throughout the engagement
- Introduced a **structured bug reporting and feedback system** to improve dev handoff, prioritise critical issues, and close feedback loops within sprints
- Set up **dedicated testing cycles per sprint**: smoke and regression testing after each deployment, targeted retesting of resolved bugs
- Delivered **cross-platform coverage** across Windows, macOS, and iOS using **BrowserStack** for cross-browser validation

---

## Results
- **100% of blockers and critical bugs caught before release**
- **0 post-release critical defects**
- **80% of submitted UI/UX improvements approved and implemented**
- Noticeable boost in user satisfaction and usability reported post-launch

---

## What Made the Difference
Embedding QA into the release cycle from day one — not as an afterthought — gave the dev team a reliable feedback loop every sprint. A living checklist and structured bug reports meant nothing fell through the cracks, even without automation.

---

## Relevant for Jobs With:
- Manual QA for AI-powered SaaS products
- QA process setup from scratch
- Exploratory and functional testing
- Cross-browser / cross-device compatibility testing (BrowserStack)
- Regression and smoke testing within sprint cycles
- Early-stage startups needing QA structure without automation
- UI/UX feedback and usability testing
