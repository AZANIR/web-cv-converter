# Case Study: 4000+ API Tests Automated & 600+ Bugs Reported in Australian Booking Platform (NDA)

## Overview
- **Client type:** Hospitality tech platform
- **Industry:** Travel & Hospitality / Booking Technology (Australia)
- **Service provided:** API Tester / QA Engineer — full automation setup, CI pipeline, documentation improvement
- **Tags:** API Testing, Test Automation, JavaScript, Postman, Newman, Allure, CI/CD, Regression Testing, Bug Reporting, Documentation, SaaS, QA

---

## The Challenge
An Australian hospitality tech platform had no structured API testing process, no automation, and no reliable reporting. Integration testing was being done manually by developers with no formal test documentation — leaving critical issues undetected and slowing releases.

Key pain points:
- No automated test coverage despite Swagger (OpenAPI) documentation being available
- Integration checks performed manually by developers — prone to human error and missed edge cases
- No formal test reporting or visibility into API health
- Inaccurate and incomplete API documentation causing ambiguity
- No test data management — stale or inconsistent data creating unreliable test results

---

## What I Did
- Led the creation and automation of **4,000+ API test cases** using **JavaScript and Postman**, covering smoke, regression, and integration scenarios across both staging and production environments
- Set up **Newman** for automated test execution and **Allure** for rich, environment-specific reporting dashboards — giving QA and developers independent, fast feedback loops
- Built a full **integration testing pipeline** verifying API behavior and interactions across all system components
- Implemented **automated creation and cleanup of test data** to ensure endpoint consistency and surface bugs related to endpoint interaction
- Improved API documentation by organising related issues in GitLab, clarifying endpoint behaviour, and collaborating with developers and the client to fill in missing details
- Integrated **Gmail and Outlook** environments for testing notification flows end-to-end

---

## Results
- **4,000+ test cases automated**, significantly reducing regression time and manual QA effort
- **600+ high-quality bug reports submitted**, accelerating issue resolution across the team
- Full integration testing pipeline established — all API interactions verified across system components
- Stable, automated coverage gave the client confidence in every production release
- Improved documentation became a shared reference, reducing developer ambiguity and back-and-forth

---

## What Made the Difference
Fixing documentation and test data management before scaling automation was the foundation. Once endpoints were clearly defined and test data was reliable, automation compounded quickly — producing trustworthy results rather than noisy, flaky tests that developers would start ignoring.
