# Case Study: 220+ Bugs Reported & UX Improved in U.S. Real Estate Web App (NDA)

## Overview
- **Client type:** Real estate web platform
- **Industry:** Real Estate / PropTech (USA)
- **Service provided:** Manual QA Engineer + Load Testing — full QA strategy, cross-platform testing, performance validation
- **Tags:** Manual Testing, Exploratory Testing, Functional Testing, Cross-Browser Testing, Mobile Testing, Bug Reporting, UX Testing, Performance Testing, k6, Real Estate

---

## The Challenge
A U.S.-based mobile web platform for renting, buying, and selling homes had no QA involvement, no testing documentation, and no formal specs — only UI mockups in Adobe XD. Critical user flows including bookings, payments, and agent interactions were entirely untested.

Key pain points:
- No test cases, checklists, or QA documentation of any kind
- No cross-browser or cross-device coverage
- Payment and booking flows untested — high risk for conversion-breaking bugs
- No performance or load testing — scalability under real user traffic unknown
- Tight two-month deadline to reach release-ready stability

---

## What I Did
- Applied **experience-based and exploratory testing** to quickly uncover flaws without formal specs, building detailed checklists for every browser and platform combination
- Tested manually across **Chrome, Firefox, and Safari** (5 previous versions each) and on real **Android and iOS devices** for full cross-platform coverage
- Reported **220+ issues** — many classified as Critical or Major — covering broken user flows, payment failures, booking disruptions, and mobile layout inconsistencies
- Collaborated closely with client and dev teams, refining test checklists iteratively throughout the engagement
- Supported every release with **regression testing**, ensuring previously resolved bugs stayed fixed
- Conducted **performance and stress testing using k6** to simulate user loads and validate system scalability under pressure

---

## Results
- **220+ bugs reported**, many Critical or Major — dramatically improving platform stability ahead of release
- Smoother flows for customers, agents, and admins across all key journeys
- Reliable call booking and payment processes after targeted fixes
- Mobile layout inconsistencies resolved across Android and iOS
- Improved conversion rates through iterative UX enhancements
- Platform went from unstable and undocumented to release-ready within two months

---

## What Made the Difference
With no specs to work from, experience-based and exploratory testing was the only viable path — and it uncovered issues that formal scripted testing would have missed entirely. Pairing that with k6 load testing gave the client confidence not just in functionality, but in how the platform would hold up under real traffic.
